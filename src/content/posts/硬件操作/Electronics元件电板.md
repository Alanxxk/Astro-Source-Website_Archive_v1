# 2.控板编程
## 2.1.原理
1. 作用：控制电路实现控制目标元件，尽量快稳准
2. 芯片：
   - CPU：解释计算机指令和处理计算机数据（计算机一部分）
   - GPU：绘图运算（计算机一部分）
   - NPU：专用于人工神经网络硬件加速（计算机一部分）
   - MPU：解释计算机指令和处理计算机数据的微型芯片（计算机一部分）
      1. 系统：Linux，Android
   - MCU：集成电路单芯片微型控制器（基本是完整计算机）
      1. 特点：体积小，耗电低，电路简单，价格便宜。但运行频率低
      2. 寄存器：二进制寄存器存储数据
         - 4位：简单家电控制
         - 8位/16位：复杂家电控制
         - 32位：使用嵌入式操作系统控制
      3. 系统：无，FreeRTOS
      4. 编译：高级编程语言不能直接在单片机中执行，需要编译转换成二进制的机器语言。C语言效率高代码精简，因而成为最适合语言
      5. 烧录：二进制的机器语言可执行文件传输到单片机芯片上
      6. 历史：
         - 单片机只能烧录一次，使用大电流把晶体管烧断
         - 使用紫外线和激光擦除烧录数据
         - 晶体管更换为电容，可以擦除和烧录多次
3. 端口：
   - VCC：整个电路工作的供电高电压
   - VDD：设备的工作供电高电压
   - VEE：设备工作的供电低电压/负电压
   - VSS：设备工作的供电公共低电压，一般是0V
   - CLK：Clock时钟信号，各种设备都按照这个固定周期脉冲信号同步频率、处理数据、数据传输
   - RST：Reset复位，重新开始程序
   - GND：Ground地线，0V
4. 英文：
   - Pin：引脚
   - Periphera：外部设备
5. 控环：（基本是完整计算机）
   - 开环控制：无反馈值，易受干扰
   - 闭环控制：有反馈值，能抗干扰，输出精度高，但需要编码传感器检测
      1. PID控制：
         - 原理：输入→(反馈测量值)→比例、积分、微分调节→执行机构→(测量)→输出
         - 控制：角度，转速，扭矩，温度，电压，电流……
         - 数值：
            1. 目标值：目标位置
            2. 反馈值：测量位置
            3. 输出量：U=控制力
            4. 误差：E=目标位置-测量位置
            5. 稳态误差：在近似稳定后，目标位置和测量位置仍存在的差值
         - 公式：U(k)=Kp·E(k)+Kd·\[E(k)-E(k-1)]+Ki·ΣE(i)，k是离散时间
            1. E：E(k)=目标值-本次测量值，E(k-1)=目标值-上一次测量值
            2. Kp·E(k)：对于位置的线性控制。Kp系数过大导致在目标位置附近振荡(超调)，Kp系数过小导致到达目标位置速度太慢和抗阻力能力太差
            3. Kd·\[E(k)-E(k-1)]：对于速度的阻尼控制。在Kd系数过大而阻力过大时，可以抑制在目标位置附近振荡(超调)但到达目标位置的速度太慢；在Kd系数过小而阻力过小时，无法抑制在目标位置附近振荡也会导致到达目标位置的速度太慢
            4. Ki·ΣE(i)：对于位置的最终细微控制(稳态误差控制)。在目标位置附近时，Kp·E(k)和Kd·\[E(k)-E(k-1)]的数值很小导致很难精确达到目标位置，添加参数是根据次数累加误差值从而辅助快速到达目标位置。但Ki系数过大又会导致在目标位置附近振荡(超调)

6. 通信：
   - USART
   - I2C/IIC
   - SPI
   - CAN
7. 引脚：
   - 名称：GPIO=General Purpose Input/Output=通用输入输出
   - 参数：
      1. 上下拉
         - NO Pull-Up-Down：不启用内部上拉下拉电阻，但引脚会悬空容易受到干扰，需要外部电路保持电平稳定
         - Pull-Up：启用内部上拉电阻，在引脚悬空时电阻拉到高电平
         - Pull-Down：启用下拉电阻，在引脚悬空时电阻拉到低电平
      2. 输入
         - ：上拉输入。在芯片引脚没有输入时，读取引脚是高电平
         - ：下拉输入。在芯片引脚没有输入时，读取引脚是低电平
         - ：浮空输入。在芯片引脚没有输入时，读取引脚是不确定电平
         - ：模拟输入。芯片引脚可以输入模拟/离散数字信号
      3. 输出
         - Push-Pull：推挽输出。强力直接驱动输出高/低电平（推挽输出电平遇到相反电平会烧坏）
         - Open-Drain：开漏输出。只能驱动低电平，而高电平需要依赖外部上拉电阻(电阻越小输出切高电平越快)，在不驱动低电平时引脚处于悬空状态。常用在I2C总线
         - ：复用推挽输出
         - ：复用开漏输出
      4. 电平
         - Low：默认输出低电平
         - High：默认输出高电平
      5. 出速
         - Low：引脚输出电平变化速度是低速，可以减少电磁干扰和功耗，一般是LED或继电器
         - Medium：中速输出，一般应用
         - High：高速输出，一般是驱动数据总线或PWM信号
         - Very High：超高速输出，一般是高速数字通信或高频信号
      6. 数模
         - ADC：外部模拟信号转换为数字信号
         - DAC：数字信号转换为外部模拟信号
8. 中断
   - 方式：
      1. 轮询：每隔一段时间访问端口查看信号，有则触发中断
      2. 打断：在接收到信号后触发中断
   - 优先级：
      1. 作用：在多个中断同时执行时，根据中断的抢占优先级数值而插队和响应优先级数值而排队
      2. 数值：数字越小而优先级越高
      3. 触发：只比自己数字小的优先级才会触发中断
9.  定时器
   - 原理：检测时钟源产生的等距方波信号的上升沿或下降沿进行计数
   - 作用：
      1. 计时（总方波数×时间间隔）
      2. PWM（一个周期中高低电平比例,在高频时表现是平均值）（通过比较计数值实现高低电平切换）
      3. 传输数据的时间间隔
   - 寄存器：
      1. PSC分频倍数：接收到的定时器信号频率太高，所以分频后开始计数
      2. CNT计数器：每次经过频率周期则计数
      3. ARR计数总数：超参数，计数总数和计数频率确定计数周期
   - 时钟源：
      1. 芯片内部：芯片内部RC振荡电路（电容电感充放电电路,容易受到环境影响而不稳定）
         - HSI：高速内部时钟，频率不稳定
         - LSI：低速内部时钟，频率不稳定，看门狗
      2. 外部晶振：使用两个引脚连接芯片（石英晶体制成晶体振荡器）
         - HSE：高速外部时钟，频率稳定
         - LSE：低速外部时钟，频率稳定
   - 倍分频：时钟源产生的频率信号，可以自主选择是否经过PLLCLK的倍分频(×÷)得到SYSCLK信号，SYSCLK信号经过AHBP的倍数分频(÷)后得到HCLK信号给主要内部设备使用（主要的时钟总线），HCLK经过AHBP2的倍分频(×÷)可以得到PCLK信号给TIM定时器、通信时钟波特率、其他外设使用
   - 通道：一个定时器有多个输出通道
   - 输出：（N是取反）
      1. 输出比较
      2. PWM生成
      3. 强制输出
10. 串口通讯
   - 速度：按bit位收发数据
   - 端口：Tx发送，Rx接收，GND地线
   - 接线：Tx接Rx，Rx接Tx，GND接GND
   - 通信：串行异步全双工
      1. 串行：传输使用一根线，每次传输一位
      2. 并行：传输使用多根线，每根线每次传输一位
      3. 同步：需要收发双方时钟频率一致，需要添加一根时钟线发送脉冲实现同步
      4. 异步：规定传输波特率而双方使用各自时钟，使用数据帧的起始位和停止位实现同步
      5. 单工：单向传输
      6. 半双工：双向传输，但双方不能同时发送/接收
      7. 全双工：双向传输，双方可以同时发送/接收
   - 信号：
      1. 高电平：1
      2. 低电平：0
   - 参数
      1. 波特率：传输bit位数据/秒，推荐921600
      2. 数据帧：
         - 起始位：1位低电平（高变低）
         - 数据位：0-7位高/低电平数据
         - 校验位：根据数据位计算的结果发送1位高/低电平，接收方依此判断传输无误。常见有无校验，奇校验，偶校验
         - 停止位：一般是1位高电平
   - 传输：
      1. CPU：寄存器→CPU(占用资源)→传输数据线
      2. DMA：寄存器→DMA外部设备→传输数据线
11. CAN通讯
   - 接线：两根缠绕线（相互抵消外界干扰），红对红（CANH,正极），黑对黑（CANL,负极）
   - 通信：串行异步半双工
   - 方向：
      1. 同向：在缠绕线捋直后，两端的端子卡扣是同向
      2. 反向：在缠绕线捋直后，两端的端子卡扣是反向
   - 传输：
      1. CAN节点：设备
      2. CAN总线：CANH，CANL
   - 信号：
      1. 类型：差分信号
      2. 隐性电平：在CAN节点的High线-Low线=0时（浮空产生），输出信号=1
      3. 显性电平：在CAN节点的High线-Low线>0时（有电压差），输出信号=0
      4. 总线电平：在总线同时存在显隐电平时，显性电平会覆盖隐性电平，可以用来确定通信设备优先级
   - 帧：
      1. 数据：多个字节 
      2. 分类：
         - 数据帧：帧起始(1位)+仲裁场(12位)+控制场(6位)+数据场(0-8字节)+CRC场(16位)+应答场(2位)+帧结尾(7位)
            1. 仲裁场：12位ID标识符。ID越小→0越多→显性电平越多→优先级越高
            2. 控制场：数据场的数据长度
            3. CRC场：校验，总线控制
         - 遥控帧
         - 错误帧
         - 过载帧
         - 帧间隔
   - 软件过滤器：
      1. 作用：设备总是判断总线信号与自身有关会浪费时间，可以通过CAN设备手册的ID和反馈报文标识符(设备反馈给芯片)和控制报文标识符(芯片控制设备)辅助区分信号（电机可以调整拨码开关可以调整ID从而影响标识符）
      2. 模式：
         - 列表模式：制作ID表，数据ID在表中才接收
         - 掩码模式：通过设置掩码，掩码和数据每位独立相乘得到的结果就是屏蔽码，使用屏蔽码和验证码比较一致则成功
   - 硬件外设：
      1. 发送：邮箱
      2. 接收：队列(FIFO=First-In-First-Out)
## 2.2.控板类型
1. Arduino
   - 软件：Arduino
   - 硬件：Arduino板，专用USB串口线
2. 51单片机
   - 软件：Keil，STC-ISP烧录软件（代码写入硬件芯片），CH340-CH341驱动
   - 硬件：51单片机，通用USB串口线
3. ESP32
   - 厂家：Espressif乐鑫（芯片是32位的MCU,单核双核）
   - 特点：芯片集成无线通信模块
4. STM32
   - 厂家：STMicroelectronics意法半导体
   - 代号：
      1. 家族：
         - STM8：8位的MCU,单核
         - STM32：32位的MCU/MPU,单核
      2. 产品：
         - A：汽车级
         - F：基础型
         - L：超低功耗
         - S：标准型
         - WB：无线产品
         - H：高性能
         - G：主流型
      3. 功能：
         - STM8系列：
            1. 103：入门级
            2. F52：汽车级
            3. L31：低端汽车级
         - STM32系列：
            1. 051：入门级
            2. 103：基础型
            3. 303：升级版，更多外设
            4. 407：高性能
            5. 723：超强性能
            6. 152：超低功耗
      4. 引脚数，闪存容量，芯片封装，温度范围……
   - 软件：VScode(或Keil)，STM32CubeMX（图形化配置芯片引脚生成代码,否则使用标准库或HAL库自行配置）
   - 硬件：STM32控制板（开发板原理图和PCB图(可能涉及设计秘密而较难获取)），通用USB串口线，烧录器（有线≈50￥(STLINK,JLINK)/无线≈300￥(正点原子)），杜邦线
   - STM32CubeMX使用：
      1. 选择芯片：在STM32CubeMX中新建项目，选择芯片型号MCU，开启项目
      2. 查原理图/用户手册：查看开发板的引脚作用，有无外部时钟源（一般是OSC），定时器输出通道引脚，串口TxRx引脚，CANTxRx引脚
      3. 调试类型：在系统核心的SYS选择串口调试
      4. 时钟类型：在系统核心的RCC选择外部晶振或外部设备提供的时钟源
      5. 配时钟树：一般是HCLK填写最大值后按下回车，自动配置时钟树，改写晶振值和选用外部晶振
      6. 配置引脚：在系统核心的GPIO选择引脚，每个引脚可以选择特定功能，选择功能配置
      7. 配定时器：在系统核心的RCC选择外部时钟源的晶振来源。在定时器选择通道输出和引脚，（频率周期=）输入PSC分频倍数(÷)(分频倍数=数值-1)和CM计数方向和ARR计数总数。在时钟树中输入外部晶振频率，选择时钟源和倍分频
      8. ***配置串口：在的选择异步，输入波特率和数据帧的数据位+校验位的位数和停止位位数。在的DMA选择添加发送接收和勾选开启全局中断和自主选择开启循环模式***
      9. 配置CAN：在连接外设的CAN勾选激活，根据CAN设备配置相应波特率(根据公式配置(RSJW越大越不容易出错但速度变慢,一般选1)(其他参数有设置先后顺序))，使能AR自动重传和TFP发送队列优先级，勾选RX中断
      10. 保存项目：点击项目管理，输入项目名称，英文路径文件夹，选择Cmake编译（使用Keil则使用MDK-ARM编译），选择下载已配置的库文件，勾选把功能代码分散在不同文件中，生成代码
   - VScode配置：
      - CubeMX后：点击左下角的Debug，选择Debug，按下F7按键编译
      - VScode插件：cmake，ninja，gcc/g++
      - 配置文件：.vscode，.clang-format，.gitignore，openocd.cfg
      - 存储文件：在CmakeLists.txt同级新建文件夹，放入自己写的代码文件
      - 代码文件：引用头文件(一般main.h)，函数前添加CPP转换成C类型编译(externC)，函数内部调用HAL库函数
      - 主函引码：在main.c文件中的main函数外externC函数声明，在main函数中的User_Code注释夹层中调用函数(否则在重新生成时会被覆盖)
      - Cmake配置：在CmakeLists.txt的include_directories添加代码文件夹名称，并在sources添加代码文件夹名/文件名的路径
      - 文件编译：<kbd>F7</kbd>
      - 文件烧录：<kbd>CTRL</kbd>+<kbd>Shift</kbd>+<kbd>B</kbd>
      - 代码调试：<kbd>F5</kbd>，在CORTEX-LIVE-WATCH添加需要观察的变量

## 2.3.STM32控制编程
1. 头文件固定代码：避免头文件重复命名导致重复编译而失败
   - #ifndef：如果宏没有定义
   - #define：定义宏
   - #endif：结束判断
2. 基础功能函数（时间单位ms）
   - 延时：HAL_Delay(时间值)（阻塞整个CPU）
   - GPIO：
      1. 翻转：HAL_GPIO_TogglePin(GPIO引脚分组（Px:x是组别）,GPIO_PIN引脚)
   - 定时器：
      1. 初始化：
         - 开启定时器：HAL_TIM_PWM_Start(&htim定时器,TIME_CHANNEL定时器通道)
      2. 使用：
         - 设置比较值：__HAL_TIM_SetCompare(&htim定时器,TIME_CHANNEL定时器通道,高低电平切换的比较值)
   - 串口：
      1. 数据帧：变量
         - 1字节：uint8_t，int8_t，char
         - 2字节：uint16_t，int16_t
         - 4字节：uint32_t，int32_t，int，uint，float(fp32)
         - 8字节：double(fp64)
      2. CPU收发数据帧：
         - 接收：HAL_UART_Receive(&串口号,&数据帧变量,字节数,超时时间)
         - 发送：HAL_UART_Transmit(&串口号,&数据帧变量,字节数,超时时间)
      3. DMA收发数据帧：
         - 接收：HAL_UART_Receive_DMA(&串口号,&数据帧变量,字节数)
         - 发送：HAL_UART_Transmit_DMA(&串口号,&数据帧变量,字节数)
      4. 串口接收中断函数：
         - 接函：***HAL_UART_RxCpltCallback()***
   - CAN:
      1. 发送：HAL_CAN_AddTxMessage(&CAN号,&CAN发送结构体(写入部分参数),CAN数据帧的数组数据,&CAN邮箱变量(32位空值))
      2. 接收中断函数：HAL_CAN_RxFifo0MsgPendingCallback(CAN_HandleTypeDef *CAN号)
      3. 接收：HAL_CAN_GetRxMessage(CAN号, CAN_RX_FIFO0,&CAN接收结构体,CAN数据帧的数组数据)
3. LED和显示屏
   - 作用：合理控制显示元件，可以知道目前状态
4. FreeRTOS
   - 作用：多个任务近似同时进行(不断快速轮着切换任务)
   - 优先级：
      1. 分类：
         - 高优先级
         - 低优先级
      2. 特性：
         - 抢占：在高优先级任务触发时，直接抢占CPU而挂起暂停低优先级任务
         - 同级：在多个相同优先级任务时，按照任务就绪先后执行，每达到一个时间片会自动轮询任务
   - 任务状态：
      1. 就绪：任务等待分配CPU
      2. 阻塞：任务不能使用CPU
   - 配置方法：
      1. 工具：在STM32CubeMX中，选择中间件和软件包的FreeRTOS
      2. 版本：FreeRTOS选择CMSIS_V1版本，打开任务与队列
      3. 参数：添加任务，输入任务名称和任务的栈的字节数和任务函数名称，选择任务优先级和生成任务函数声明(Default是生成函数框架,AsExternal是生成函数声明,AsWeak是生成空的虚函数(写同名函数会覆盖这个虚函数作用))
   - 代码解析：
      1. 任务句柄：≈任务名称
      2. 函数声明：STM32CubeMX自动在FreeRTOS.c声明任务函数，而根据设置在FreeRTOS.c或其他文件手动定义函数
      3. 创建任务：STM32CubeMX会自动在MX_FREERTOS_Init中根据任务参数去创建任务得到任务句柄
      4. 定义任务：自己创建.c和.h文件声明和定义任务函数
   - 功能函数：来源FreeRTOS.h
      1. 阻塞延时：vTaskDelay(时间值)
      2. 进入系统：MX_FREERTOS_Init()（不再执行主函数Main中的While）
